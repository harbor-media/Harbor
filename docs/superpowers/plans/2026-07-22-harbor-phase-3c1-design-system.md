# Harbor Phase 3c-1 — Design System Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace Harbor's navy/violet theme with the operator's achromatic shadcn design system, without changing any behavior.

**Architecture:** New tokens are added alongside the old ones, pages are converted a pair at a time, and the legacy tokens are deleted only once nothing references them. That ordering keeps the app coherent at every commit, which matters because the Playwright suite runs against each task.

**Tech Stack:** Tailwind 4.3.3, shadcn (Vite), Radix primitives, lucide-react, class-variance-authority, clsx, tailwind-merge, bundled Geist / Geist Mono / Outfit.

**Spec:** `docs/superpowers/specs/2026-07-22-harbor-phase-3c1-design-system-design.md`

## Global Constraints

- **Never add `Co-Authored-By` trailers, "Generated with Claude Code" footers, or any AI attribution to any commit message or PR body.** Absolute rule.
- Work on the phase branch, never `main`. Confirm with `git branch --show-current` before the first commit of every task.
- `apps/web` uses `bundler` module resolution — **imports are extensionless**. `packages/*` and `apps/server` use `nodenext` and need explicit `.js`. Mixing breaks the build.
- **No behavior changes.** Same routes, same requests, same states, same copy.
- **Every existing test must pass unchanged.** A test that needs editing to accommodate the restyle is evidence behavior changed — reconsider the change rather than rewriting the test.
- **Fonts must be bundled, never linked from a CDN.** Harbor's CSP sets `font-src 'self'`; a Google Fonts link is blocked and silently falls back to system sans.
- **No `image/svg+xml`, no external `<link>` to any third-party origin.** The CSP forbids it and self-hosting is the point.
- Dependencies are pinned to exact versions, no `^` or `~`, matching the rest of the repo.
- Every input keeps an associated `<label>`; error regions keep `role="alert"` and `aria-live`; focus states stay visible. `CLAUDE.md` targets WCAG 2.2 AA.
- `no-console` is an ESLint error.
- Every task ends green on `pnpm lint`, `pnpm typecheck`, `pnpm test`, `pnpm build`.

- [ ] **Step 2: Add the path alias**

shadcn generates components that import from `@/lib/utils`. Without this alias those imports fail to resolve.

`apps/web/tsconfig.json` — add inside `compilerOptions`:

```json
    "baseUrl": ".",
    "paths": { "@/*": ["./src/*"] }
```

`apps/web/vite.config.ts` — add the matching runtime alias, since TypeScript paths do not affect the bundler:

```ts
import path from "node:path";
// ...
export default defineConfig({
  plugins: [react(), tailwindcss()],
  resolve: {
    alias: { "@": path.resolve(import.meta.dirname, "./src") },
  },
  base: "/",
  // ...rest unchanged
});
```

- [ ] **Step 3: Create the class-merge helper**

`apps/web/src/lib/utils.ts`:

```ts
import { clsx, type ClassValue } from "clsx";
import { twMerge } from "tailwind-merge";

/**
 * Merges class names and resolves Tailwind conflicts, so a caller-supplied
 * `className` can override a component's defaults instead of both landing in
 * the class list and the winner being decided by stylesheet order.
 */
export function cn(...inputs: ClassValue[]): string {
  return twMerge(clsx(inputs));
}
```

- [ ] **Step 4: Replace the theme**

`apps/web/src/index.css` — replace the whole file. The legacy `harbor-*` and `accent-*` tokens are kept for now under a clearly marked block so the existing pages keep rendering; Task 7 deletes them once nothing refers to them.

```css
@import "tailwindcss";

/* Bundled, never CDN: Harbor's CSP sets font-src 'self', so a Google Fonts
   link is blocked and falls back to system sans -- a silent failure that
   presents as "the fonts didn't apply". Self-hosting also means no
   third-party request announcing when someone opens their media server. */

@import "@fontsource-variable/geist-mono";
@import "@fontsource-variable/outfit";

@custom-variant dark (&:where(.dark, .dark *));

@theme {

     token. Color appears only where it carries meaning, which lets poster
     artwork supply all the color on a catalog screen. */

  --color-foreground: oklch(0.985 0 0);
  --color-card: oklch(0.225 0 0);
  --color-card-foreground: oklch(0.985 0 0);
  --color-popover: oklch(0.225 0 0);
  --color-popover-foreground: oklch(0.985 0 0);
  --color-primary: oklch(0.922 0 0);
  --color-primary-foreground: oklch(0.205 0 0);
  --color-secondary: oklch(0.285 0 0);
  --color-secondary-foreground: oklch(0.985 0 0);
  --color-muted: oklch(0.285 0 0);
  --color-muted-foreground: oklch(0.715 0 0);
  --color-accent: oklch(0.285 0 0);
  --color-accent-foreground: oklch(0.985 0 0);
  --color-border: oklch(1 0 0 / 10%);
  --color-input: oklch(1 0 0 / 15%);
  --color-ring: oklch(0.556 0 0);

  /* Semantic status only. */
  --color-destructive: oklch(0.704 0.191 22.216);
  --color-success: oklch(0.696 0.17 162.48);
  --color-warning: oklch(0.828 0.189 84.429);
  --color-info: oklch(0.746 0.16 232.661);

  --font-sans: "Geist Variable", ui-sans-serif, system-ui, sans-serif;
  --font-mono: "Geist Mono Variable", ui-monospace, monospace;
  --font-display: "Outfit Variable", ui-sans-serif, system-ui, sans-serif;

  --radius: 10px;
  --radius-sm: calc(var(--radius) * 0.6);
  --radius-md: calc(var(--radius) * 0.8);
  --radius-lg: var(--radius);
  --radius-xl: calc(var(--radius) * 1.4);
  --radius-2xl: calc(var(--radius) * 1.8);

  /* LEGACY -- deleted in Task 7 once no page references them. Kept only so
     pages restyled in later tasks keep rendering in the meantime. */

  --color-harbor-900: oklch(0.225 0 0);
  --color-harbor-800: oklch(0.285 0 0);
  --color-accent-500: oklch(0.922 0 0);
  --color-accent-400: oklch(0.922 0 0);

body {
  background-color: var(--color-background);
  color: var(--color-foreground);
  font-family: var(--font-sans);

h1,
h2,
h3 {
  font-family: var(--font-display);
}
```

Note the legacy tokens are remapped onto the **new** greyscale rather than left navy. That way the palette flips in one commit and each later task is a structural conversion, not a colour change you have to eyeball twice.

### Task 3: Convert Login and Setup

**Files:**
- Modify: `apps/web/src/pages/Login.tsx`, `apps/web/src/pages/Setup.tsx`

**Interfaces:**
- Consumes: `Button`, `Input`, `Label`, `Card`, `Alert` from `@/components/ui/*`.

Read both files before editing. Convert markup to the primitives and the class names to the new tokens using this mapping, which applies to every page task:

| Old | New |
| --- | --- |
| `bg-harbor-950` | `bg-background` |
| `bg-harbor-900` | `bg-card` |
| `bg-harbor-800` | `bg-secondary` |
| `text-accent-500` | `text-foreground` (headings) or `text-primary` (emphasis) |
| `bg-accent-500` | `bg-primary` with `text-primary-foreground` |
| `rounded-card` | `rounded-xl` |
| `font-display` | keep — now maps to Outfit |
| ad-hoc `text-red-400` | `text-destructive` |
| `focus:ring-accent-500` | `focus-visible:ring-ring` |

**Behavior that must not change:**

- Every input keeps its `id` and its `<label htmlFor>`. The Playwright suite finds fields by `getByLabel("Username or email")`, `getByLabel("Password")`, `getByLabel("Server name")`, `getByLabel("Username")`, `getByLabel("Email")`. Change a label string and the suite breaks — which is the point of the net, but it means the copy stays as-is.
- Buttons keep their accessible names: `/sign in/i`, `/create owner/i`.
- Error paragraphs keep `role="alert"` and `aria-live="assertive"`. Prefer wrapping them in `Alert` with `variant="destructive"` while preserving those attributes.
- `Setup.tsx` keeps `const USERNAME_PATTERN = "^[a-zA-Z0-9._\\-]+$"` **exactly**. The hyphen escape is load-bearing: browsers compile the `pattern` attribute with the RegExp `v` flag, and an unescaped trailing hyphen makes the whole attribute invalid, silently disabling validation.

### Task 8: Update CLAUDE.md

**Files:**
- Modify: `CLAUDE.md`

- [ ] **Step 1: Rewrite the Visual Direction section**

Replace the "Brand direction" bullets — currently "Deep navy surfaces" and "Blue and violet accent gradients" — with the adopted system:

- Achromatic OKLCH surfaces; chroma is zero on every surface and text token.
- One emphasis color, Signal Off-White `oklch(0.922 0 0)`. On dark, high-contrast near-white *is* the accent.
- Color is semantic, never decorative: red for error, green for success, amber for warning, sky for information. A screen with nothing wrong is essentially greyscale, and that is correct.
- Never pure black or pure white. The canvas is `oklch(0.185 0 0)`, text `oklch(0.985 0 0)`.
- Hairline borders `oklch(1 0 0 / 10%)` and tonal layering instead of shadows.
- Outfit for display, Geist for body, Geist Mono for labels and values that read as data.
- Large cinematic artwork carries the color. Neutral chrome exists so posters are the only saturated thing on screen.

Keep the existing "Design Rules" list — it is unchanged by this phase, and it already says minimal visual noise and no gradients everywhere.
